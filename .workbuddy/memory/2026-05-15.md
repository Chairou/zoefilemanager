# 2026-05-15 工作日志

## 修复：install_name_tool 签名失效 warning + 签名验证失败

构建时 macOS install_name_tool 输出 6 条 warning：
> changes being made to the file will invalidate the code signature in: libsmbclient.0.8.1.dylib

且 `codesign --verify --deep --strict` 报：
> a sealed resource is missing or invalid: file added: .../Contents/.macdeployqt.done

### 根因

1. **warning**：Homebrew 的 dylib 自带 ad-hoc 签名；install_name_tool 修改 LC_LOAD_DYLIB 时，旧签名 hash 不再匹配 → 输出 warning。warning 本身无害（我们后续会重签），但日志噪声大。
2. **签名验证失败**：原本 sentinel 文件 `.macdeployqt.done` 是在 codesign --deep 之后才 `touch` 创建的，导致 bundle 被签后又出现新文件 → 资源不匹配。

### 修复

**1. `cmake/deploy_macos.cmake`**
- 新增 `_strip_signature(path)` 辅助函数（`codesign --remove-signature`）
- 在每个 `install_name_tool -id / -change / -add_rpath` 调用前先剥签
- 把 sentinel 创建（`file(TOUCH .../Contents/.macdeployqt.done)`）移到 codesign --deep **之前**，让 sentinel 一并被 seal

**2. `CMakeLists.txt`**
- 移除 POST_BUILD 末尾多余的 `touch sentinel` 命令（已交给 deploy_macos.cmake 在 codesign 前完成）
- macdeployqt 输出过滤里追加 grep 规则：过滤 `install_name_tool: warning: changes being made to the file will invalidate the code signature`（针对 macdeployqt 内部调用 install_name_tool 产生的同款 warning，无法剥签拦截）

### 验证

- `cmake --build build -j` 完整 deploy，零 warning
- `codesign --verify --deep --strict --verbose=2 ZoeFileManager.app`：
  - `valid on disk`
  - `satisfies its Designated Requirement`
- sentinel 跳过逻辑仍正常：第二次 build 输出 "bundle already deployed — skipping"

### 关键文件

- `cmake/deploy_macos.cmake` — `_strip_signature` 函数 + 9 处 install_name_tool 前剥签 + sentinel touch 时机
- `CMakeLists.txt` — grep 链增加 install_name_tool warning 过滤、移除多余 touch

## 长期约定（建议放 MEMORY.md）

macOS bundle deploy：
- 任何 `install_name_tool` 调用前应先 `codesign --remove-signature`，否则会输出"will invalidate"warning（虽然无害但污染日志）
- 在 bundle 内创建/修改文件必须在 `codesign --deep` **之前**，否则 `codesign --verify --deep --strict` 会失败
