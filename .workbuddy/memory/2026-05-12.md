## 00:05 dark theme 工具栏按钮"接缝色差"修复（用户截图反馈）

### 现象
用户截图框出工具栏前进/后退/上一级/刷新等按钮**之间的细缝**仍有比按钮 `#262B35` 略深的竖条色差，按钮看起来像悬浮块未与工具栏完全融合。

### 根因
`src/MainWindow.cpp` 的 `applyDarkTheme()` 中：
1. `QToolBar QToolButton` 设置了 `border: 1px solid transparent` + `border-radius: 4px` —— 透明边框 + 圆角导致按钮"实色绘制区域"内缩 1px，按钮角部因抗锯齿与工具栏底色发生像素混合，肉眼看就是接缝处一条不同色的细线。
2. `QToolBar { spacing: 4px }` + `QToolBar::separator { width: 1px; margin: 4px 2px }` —— spacing 与 separator 边距叠加，按钮之间天然留出空白条。虽然工具栏 bg 已经同色，但叠加上面 1px 圆角抗锯齿，渲染出来的接缝就有色差。
3. 缺少 `outline: none` 与 `:focus` 显式压制，mac native style 可能在某些状态下绘制额外阴影。

### 修复方案
按钮 base 态彻底"实心矩形紧贴"：
- `border: none; border-radius: 0px; outline: none; margin: 0px;`
- `padding: 6px 12px;`（撑开点击区域，替代 spacing）
- `QToolBar { spacing: 0px }` + `QToolBar::separator { width: 0px; margin: 0px }` —— 按钮与按钮完全无缝
- 新增 `QToolBar QToolButton:focus { outline: none; border: none; }`
- hover/pressed/checked 不再使用 border 颜色变化（去掉 `border-color`），仅靠 background 和文字颜色变化做交互反馈，避免 hover 时按钮"突然冒出 1px 边框"导致挤压

### 改动文件
- `src/MainWindow.cpp` — `applyDarkTheme()` 内 QToolBar / QToolBar::separator / QToolBar::handle / QToolBar QToolButton（base/focus/hover/pressed/checked）

### 编译结果
`cmake --build build -j` → 0 error / 0 warning（macdeployqt 的 qt.conf 警告为打包脚本噪声）

### 经验
QSS 中"按钮间接缝色差"的常见 root cause：透明 border + border-radius 引发的边缘抗锯齿混合 + spacing 露出底层。消除方案：base 态实心矩形（无 border/no radius）+ spacing: 0 + padding 撑开。

## 00:48 dark theme 工具栏按钮间彻底消除分隔线（用户继续微调）

### 用户诉求
"请在工具栏的按钮之间设置 border 为 0px。" —— 即按钮**之间**任何形式的分隔线/边界都不能出现。

### 根因（在已 border:0px 基础上仍可能露出 1px 分隔感）
1. QSS `border: 0px` 无法压制 mac/fusion 的 `QStyle::PE_FrameButtonTool` primitive — 它走 style hint，不受 stylesheet border 控制；
2. 某些 Qt 版本下即便 `QToolBar::separator { width: 0 }` 仍会渲染 1px 颜色条（与 background 颜色不同时尤其明显）；
3. 仅声明 `border: 0px` 单值不能完全压制 cascade 复活的四向 border 子属性（`border-left/right/top/bottom`）。

### 修复
1. **代码侧（关键）**：在 `setupToolbar()` 末尾遍历所有 QToolButton 调用 `setAutoRaise(true)`，关闭 native frame；并在 `eventFilter` 异步分支里也做同样遍历，覆盖 Qt 后置创建的扩展按钮（`qt_toolbar_ext_button`）。
2. **QSS 侧**：
   - QToolBar::separator → `background: transparent; width: 0px; height: 0px; margin: 0px; padding: 0px; border: 0px`（彻底 0/transparent）
   - QToolBar::handle → 加 `width: 0px; margin: 0px; padding: 0px`
   - QToolBar QToolButton 各状态（base/focus/hover/pressed/checked）显式补全四向 `border-left/right/top/bottom: 0px`，防止 cascade 复活

### 改动文件
- `src/MainWindow.cpp` — setupToolbar() 末尾追加 `setAutoRaise` 遍历；eventFilter 异步分支同步追加；applyDarkTheme() 内 QToolBar::separator/handle/QToolButton 全状态 QSS 增强

### 编译
0 error / 0 compiler warning。

### 经验
QSS `border: 0px` 不足以消除 QToolButton 之间的 native frame 分隔感，**必须叠加 `QToolButton::setAutoRaise(true)` 代码侧调用** —— 这是关闭 mac/fusion native PE_FrameButtonTool primitive 的唯一可靠方式。stylesheet 单值 border 还可能被 cascade 子属性复活，四向显式声明更稳。

## 01:14 Show/Hide 隐藏文件按钮不再重置目录树当前指向

### 用户诉求
> 点击 Show/Hide 按钮的时候，不要动目录树的显示的当前目录指向。

### 根因
`MainWindow::onToggleHidden()`（src/MainWindow.cpp:1640）在切换 `RealFileSystem::setShowHidden` 后调用了 `m_dirTree->buildTree()`。`DirectoryTree::buildTree()` 首行就是 `clear() + m_pathToItem.clear()`，整棵树被销毁重建并只展开 `/` —— 用户之前展开的所有节点、滚动位置、currentItem 全部丢失，随后两次 `highlightPath()` 虽然恢复高亮，但视野已经被重置到树顶。

### 修复（最小变更）
删除 `onToggleHidden()` 里的 3 行：
- `m_dirTree->buildTree();`
- `m_dirTree->highlightPath(m_leftPanel->currentPath(), PanelSide::Left);`
- `m_dirTree->highlightPath(m_rightPanel->currentPath(), PanelSide::Right);`

保留左右面板的 `refresh()` —— 即 Show/Hide 仅影响右侧两个文件视图的列表，**不触动左侧目录树**。隐藏目录的显示差异会在用户下次展开某个节点时由 `lazyPopulate()` 自然生效；当前已展开节点保持旧状态，视觉锚点稳定。

### 改动文件
- `src/MainWindow.cpp::onToggleHidden()`（1640-1655 段）

### 编译
0 error / 0 compiler warning。

### 经验沉淀
**切换 UI filter / 视图选项时，不要 rebuild 左侧导航树 —— QTreeWidget 的 `clear()` 会让所有展开状态 / 滚动位置 / 当前选中丢失，用户的"位置感"瞬间崩塌。** 只刷新受影响的视图（这里是文件列表），导航树的可视状态应作为不可变锚点保留。如果必须刷新树，应先保存 expanded-paths + currentPath + scrollValue，再 clear+rebuild，最后逐条恢复 —— 但成本显著高于"不动树"。

## 01:35 工具栏按钮 tooltip 悬停不显示 —— 修复

### 现象
用户反馈：鼠标移到工具栏按钮上，无 tooltip 提示，看不到按钮用途。

### 排查
`setupToolbar()` 里每个 QAction 都已调用 `setToolTip(T("..."))`，`retranslateUi()` 里也重新设 tooltip。按道理 Qt 会把 action->toolTip() 同步到内部 QToolButton，但实际不显示。

### 真实根因（关键经验）
**Qt 在以下组合下不会自动把 QAction::toolTip() 复制到 QToolButton::toolTip()：**
1. `toolbar->addAction(icon, "", ...)` —— action **text 为空**
2. toolbar 被 QSS 样式化（自定义 QToolBar QToolButton {...}）后进入 styled 模式
3. `setToolButtonStyle(Qt::ToolButtonIconOnly)` 时 Qt 依赖 button 自身 toolTip 而不是 action 的

这三个条件本项目全中，所以 action tooltip 始终留在 action 层，没到 button 层，鼠标悬停看不到。

### 修复
新增 `MainWindow::syncToolbarButtonToolTips()`：遍历 toolbar 所有 QToolButton，把 `btn->defaultAction()->toolTip()` 显式 `btn->setToolTip(...)`。调用点三处：
1. `setupToolbar()` 末尾（初始装配后）
2. `retranslateUi()` 末尾（语言切换后）
3. `eventFilter` 异步分支（布局变化 / 溢出按钮后置创建）

### 沉淀
**Qt QSS 样式化 toolbar + action text 空 → 必须代码层手动把 action tooltip 同步到 button tooltip；QAction::toolTip 不会自动到位。**
这条经验同 `setAutoRaise(true)` 消除 native frame、`addSeparator()` 必须代码删除，同属 "QSS 压不住 Qt style primitive / widget behavior，必须代码兜底" 系列。

### 改动
- `src/MainWindow.h` +1 方法声明 `syncToolbarButtonToolTips()`
- `src/MainWindow.cpp` +1 方法实现；三处调用点接入
- 编译：`cmake --build build -j` → 0 error / 0 warning
